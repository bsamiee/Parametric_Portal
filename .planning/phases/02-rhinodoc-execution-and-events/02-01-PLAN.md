<objective>
Build the command execution engine: protocol contract extensions for execution semantics, undo-wrapped direct API execution via CommandExecutor, RunScript command execution via ScriptRunner with EndCommand result detection, and typed RhinoCommon document facade via DocumentApi.

Purpose: The agent needs to execute arbitrary Rhino commands (RunScript) and precise RhinoCommon API calls (direct API) inside Rhino. Each write operation must be undo-wrapped so Cmd+Z reverses individual tool calls atomically. The bifurcated undo strategy (CONTEXT.md locked decision) splits behavior: direct API uses BeginUndoRecord/EndUndoRecord + AddCustomUndoEvent; RunScript creates its own undo record internally and must NOT be wrapped.

Output: Protocol contracts extended with execution-specific enums, models, and value objects. Three new execution modules (CommandExecutor, ScriptRunner, DocumentApi) ready for wiring into KargadanPlugin dispatch in Plan 02.
</objective>

    **DocumentApi.cs** -- typed RhinoCommon facade:

    Public static class with methods returning `Fin<T>`. Each method takes `RhinoDoc doc` as first parameter. No undo logic here -- that belongs in CommandExecutor.

    Methods:
    - `CreateObject(RhinoDoc doc, GeometryBase geometry, ObjectAttributes attributes)` -> `Fin<Guid>`: Calls `doc.Objects.Add(geometry, attributes)`, returns `Fin.Fail` if `Guid.Empty` returned.
    - `DeleteObject(RhinoDoc doc, Guid objectId, bool quiet)` -> `Fin<Unit>`: Calls `doc.Objects.Delete(objectId, quiet)`.
    - `ReplaceObject(RhinoDoc doc, Guid objectId, GeometryBase geometry)` -> `Fin<bool>`: Calls `doc.Objects.Replace(objectId, geometry)`.
    - `TransformObject(RhinoDoc doc, Guid objectId, Transform transform, bool deleteOriginal)` -> `Fin<Guid>`: Calls `doc.Objects.Transform(objectId, transform, deleteOriginal)`.
    - `ModifyAttributes(RhinoDoc doc, Guid objectId, ObjectAttributes attributes)` -> `Fin<bool>`: Calls `doc.Objects.ModifyAttributes(objectId, attributes, quiet: true)`.
    - `FindById(RhinoDoc doc, Guid objectId)` -> `Fin<RhinoObject>`: Calls `doc.Objects.FindId(objectId)`, returns `Fin.Fail` if null.
    - `FindByLayer(RhinoDoc doc, string layerName)` -> `Fin<Seq<RhinoObject>>`: Calls `doc.Objects.FindByLayer(layerName)`, wraps null as empty Seq.
    - `FindObjectsSince(RhinoDoc doc, uint runtimeSerialNumber)` -> `Seq<RhinoObject>`: Wraps `doc.Objects.AllObjectsSince(runtimeSerialNumber)` into Seq.
    - `AddPoint(RhinoDoc doc, double x, double y, double z)` -> `Fin<Guid>`: Constructs `new Point3d(x, y, z)`, calls `doc.Objects.AddPoint(point)`, returns `Fin.Fail` if `Guid.Empty`. This is the concrete geometry factory that proves the direct API create path end-to-end (see Warning 4 -- at least one geometry factory must be non-placeholder).

    All return types use explicit type annotations (zero `var`). Use `switch` expression for null/empty checks. Use named parameters at domain call sites.

    **ScriptRunner.cs** -- RunScript wrapper:

    Public static class. Per CONTEXT.md locked decisions: do NOT wrap in BeginUndoRecord/EndUndoRecord. RunScript creates its own undo record internally.

    Method `RunScripted(RhinoDoc doc, string commandScript, bool echo)` -> `Fin<ScriptResult>`:

    1. Subscribe to `Command.EndCommand` event BEFORE calling RunScript.
    2. Snapshot `doc.Objects.NextRuntimeSerialNumber` before execution.
    3. Call `RhinoApp.RunScript(commandScript, echo)`.
    4. Unsubscribe from `Command.EndCommand` immediately after.
    5. If `RunScript` returns false: `Fin.Fail` with "Rhino did not execute the script."
    6. If true: check captured `CommandResult` via switch expression:
       - `Commands.Result.Success`: Build `ScriptResult` with command name, result string, created object IDs from `AllObjectsSince`, and undo serial number 0 (RunScript undo serial is tracked externally via Command.UndoRedo event).
       - `Commands.Result.Cancel`: `Fin.Fail` with cancellation message.
       - `Commands.Result.Nothing`: `Fin.Fail` with "did nothing" message.
       - `Commands.Result.Failure` and all others: `Fin.Fail` with failure message.

    CRITICAL: Do NOT cache RhinoObject references across RunScript calls (research Pitfall 2). Use GUID-based re-query via `doc.Objects.FindId` or `AllObjectsSince` after execution.

    Use `[BoundaryAdapter]` attribute since this interfaces with Rhino's command system. Mark the EndCommand handler with `[BOUNDARY ADAPTER -- event subscription for command result detection]`.

    **CommandExecutor.cs** -- orchestrates execution with undo wrapping:

    Public static class with two execution paths per bifurcated strategy:

    1. `ExecuteDirectApi(RhinoDoc doc, CommandEnvelope envelope, Func<RhinoDoc, CommandEnvelope, Fin<JsonElement>> handler, Action<AgentUndoState> onUndoRedo)` -> `Fin<CommandResultEnvelope>`:
       - Extract undo description from `envelope.UndoScope` (or default "Agent Action").
       - Call `doc.BeginUndoRecord(description)` to get serial number.
       - Call `handler(doc, envelope)` to execute the operation.
       - Call `doc.AddCustomUndoEvent("Agent State Snapshot", OnAgentUndoRedo, new AgentUndoState(...))`.
       - Call `doc.EndUndoRecord(serialNumber)`.
       - On success: build `CommandResultEnvelope.Success`.
       - On failure: call `doc.Undo()` for rollback (all-or-nothing per CONTEXT.md), build `CommandResultEnvelope.Failure`.

    2. `ExecuteRunScript(RhinoDoc doc, CommandEnvelope envelope, string commandScript, bool echo)` -> `Fin<CommandResultEnvelope>`:
       - Delegate to `ScriptRunner.RunScripted(doc, commandScript, echo)`.
       - On success: build `CommandResultEnvelope.Success` with ScriptResult serialized as JSON payload.
       - On failure: build `CommandResultEnvelope.Failure`.
       - No BeginUndoRecord/EndUndoRecord wrapping (locked decision).

    3. Static `OnAgentUndoRedo` callback method:
       - MUST re-register itself via `e.Document.AddCustomUndoEvent` for redo support (CONTEXT.md locked decision).
       - MUST NEVER modify RhinoDoc (CONTEXT.md locked decision, research Pitfall 3).
       - Only reads `e.Tag` (cast to `AgentUndoState`) and re-registers.
       - MUST call the `Action<AgentUndoState>` callback captured via closure to push undo notification to EventPublisher. The callback is provided by KargadanPlugin at dispatch time and closes over EventPublisher + SessionHost identity.
       - Implementation: `ExecuteDirectApi` captures `onUndoRedo` in a local lambda that wraps `OnAgentUndoRedo` and invokes the callback. The lambda is passed to `AddCustomUndoEvent` instead of a static method reference.
       - Mark with `[BOUNDARY ADAPTER -- RhinoDoc custom undo event callback; re-registers for redo]`.

    Helper method `BuildResultEnvelope` that takes `CommandEnvelope`, `Fin<JsonElement>` result, `uint undoSerial`, and timing, and produces the appropriate Success/Failure variant. Use `ExecutionMetadata.Create` for timing and revision. Use `DedupeMetadata` with `DedupeDecision.Executed` and the envelope's `RequestId`.

    Use `[BoundaryAdapter]` attribute on CommandExecutor since it interfaces with RhinoDoc undo system.
