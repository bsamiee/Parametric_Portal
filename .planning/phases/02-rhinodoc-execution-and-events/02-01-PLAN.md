The onCommand callback currently returns `Fin.Fail("not yet implemented")`. CommandExecutor replaces this stub.

Create `apps/kargadan/plugin/src/execution/CommandExecutor.cs` (~250-300 lines). This single file contains the FULL execution engine: undo-wrapped direct API execution, RunScript with result tracking, and direct RhinoCommon API facade methods. NO separate files for ScriptRunner or DocumentApi — density is genuine polymorphism over the two execution modes.

**File structure (section separators):**

```
// --- [TYPES] -----------------------------------------------------------------
// --- [CONSTANTS] -------------------------------------------------------------
// --- [ERRORS] ----------------------------------------------------------------
// --- [FUNCTIONS] -------------------------------------------------------------
// --- [EXPORT] ----------------------------------------------------------------
```

**[TYPES] section:**
- `internal delegate void AgentStateCallback(AgentUndoState state, bool isUndo)` — callback signature for when undo/redo restores/advances agent state. CommandExecutor does not own the notification mechanism — the caller (KargadanPlugin) provides this delegate to bridge to EventPublisher.

**[CONSTANTS] section:**
- Pre-allocated static readonly Error fields for common failures:
  - `ScriptNotExecuted = Error.New(message: "Rhino did not execute the script.")`
  - `ScriptCancelled(string commandName)` — method returning Error since command name varies
  - `DocumentModificationFailed = Error.New(message: "Failed to modify document.")`
  - `ObjectNotFound(Guid objectId)` — method returning Error
  - `DirectApiOperationFailed(string operation)` — method returning Error

**[FUNCTIONS] section — three groups:**

**Group 1: Execution orchestration (public static methods)**

`ExecuteDirectApi(RhinoDoc doc, CommandEnvelope envelope, Func<RhinoDoc, CommandEnvelope, Fin<JsonElement>> handler, AgentStateCallback onUndoRedo) -> Fin<JsonElement>`:
- Extract undo scope description from `envelope.UndoScope.Map(scope => scope.Value).IfNone("Agent Action")`
- Call `doc.BeginUndoRecord(description)` — capture the returned uint serial number
- Execute `handler(doc, envelope)` to get `Fin<JsonElement>` result
- Call `doc.AddCustomUndoEvent("Agent State Snapshot", MakeUndoHandler(onUndoRedo), new AgentUndoState(RequestId: envelope.Identity.RequestId, UndoSerial: undoSerial))`
- Call `doc.EndUndoRecord(undoSerial)`
- On success: return the result payload
- On failure: call `doc.Undo()` to roll back the entire record, then return the error
- CRITICAL: The undo handler callback (`MakeUndoHandler`) must be a static method that creates an `EventHandler<CustomUndoEventArgs>` which: (a) casts `e.Tag` to `AgentUndoState`, (b) calls `e.Document.AddCustomUndoEvent(...)` to re-register for redo support (toggle pattern), (c) invokes the `AgentStateCallback` to notify harness. The re-registration uses a closure-captured reference to the callback. Use `[BoundaryImperativeExemption(Reason = "CancellationGuard", Ticket = "EXEC-05", Expiry = "2026-08-22")]` on the static handler method because the EventHandler delegate signature from RhinoCommon requires imperative void callback.

`ExecuteScript(RhinoDoc doc, string commandScript, bool echo) -> Fin<ScriptResult>`:
- Subscribe to `Command.EndCommand` event BEFORE calling RunScript
- Capture `Commands.Result` and command name in local variables
- Snapshot `doc.Objects.NextRuntimeSerialNumber` before execution to count new objects
- Call `RhinoApp.RunScript(commandScript, echo)`
- Unsubscribe from `Command.EndCommand` immediately after RunScript returns
- Do NOT wrap in BeginUndoRecord/EndUndoRecord — RunScript creates its own undo record
- Return based on RunScript boolean AND captured CommandResult:
  - `false` -> `Fin.Fail(ScriptNotExecuted)`
  - `true` + `Commands.Result.Success` -> `Fin.Succ(ScriptResult.Create(...))`
  - `true` + `Commands.Result.Cancel` -> `Fin.Fail(ScriptCancelled(commandName))`
  - `true` + any other -> `Fin.Fail(Error.New($"Command '{commandName}' failed: {result}"))`
- Use switch expression for the result dispatch (not if/else)
- CRITICAL: Object references are INVALIDATED after RunScript — do NOT hold any RhinoObject references across the call. Use `doc.Objects.AllObjectsSince(snBefore)` after to count created objects.
- `[BoundaryImperativeExemption(Reason = "ProtocolRequired", Ticket = "EXEC-01", Expiry = "2026-08-22")]` on the method because Command.EndCommand requires imperative event handler subscribe/unsubscribe and the local capture pattern is inherently stateful.

**Group 2: Direct RhinoCommon API facades (public static methods)**
These are thin Fin<T>-returning wrappers over RhinoDoc.Objects and RhinoDoc.Layers. Each method:
- Takes `RhinoDoc doc` as first parameter
- Returns `Fin<T>` where T is the typed result
- Uses switch expression on result (e.g., `Guid.Empty` -> Fail, otherwise -> Succ)

Implement these facades:
- `AddObject(RhinoDoc doc, GeometryBase geometry, ObjectAttributes attributes) -> Fin<Guid>`
- `DeleteObject(RhinoDoc doc, Guid objectId, bool quiet) -> Fin<Unit>`
- `ReplaceObject(RhinoDoc doc, Guid objectId, GeometryBase geometry) -> Fin<Unit>`
- `TransformObject(RhinoDoc doc, Guid objectId, Transform transform) -> Fin<Unit>`
- `FindById(RhinoDoc doc, Guid objectId) -> Fin<RhinoObject>`
- `FindByLayer(RhinoDoc doc, string layerName) -> Fin<Seq<RhinoObject>>`
- `ModifyAttributes(RhinoDoc doc, Guid objectId, ObjectAttributes attributes) -> Fin<Unit>`

**Group 3: Internal helpers (private static)**
- `MakeUndoHandler(AgentStateCallback onUndoRedo) -> EventHandler<CustomUndoEventArgs>` — creates the callback that re-registers and notifies. This is the redo toggle pattern from McNeel's official custom undo sample.
