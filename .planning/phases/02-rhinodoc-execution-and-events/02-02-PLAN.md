READ FIRST: Open and deeply understand EventPublisher.cs (the existing event queue pattern), the RawDocEvent and EventBatchSummary models from ProtocolModels.cs (created in Plan 01), and the EventType/EventSubtype SmartEnums from ProtocolEnums.cs (extended in Plan 01). Also read the research's Pattern 3 (Event Aggregation with Debounced Batching), Pattern 4 (Undo Detection and Harness Notification), and the SampleCsEventWatcher reference implementation link. Understand that EventPublisher.Publish accepts an EventEnvelope + Instant, and the flush callback must wrap the EventBatchSummary into an EventEnvelope via EventEnvelope.Create.

Create `apps/kargadan/plugin/src/observation/ObservationPipeline.cs` (~250-300 lines). This single file contains ALL event observation: subscription management, Channel-based aggregation, timer-driven flush, undo/redo detection, and lifecycle start/stop. NO separate files for EventSubscriber, EventAggregator, UndoObserver, or ObservationLifecycle — all four concerns compose into one cohesive module.

**File structure (section separators):**

```
// --- [TYPES] -----------------------------------------------------------------
// --- [CONSTANTS] -------------------------------------------------------------
// --- [LIFECYCLE] -------------------------------------------------------------
// --- [SUBSCRIPTION] ----------------------------------------------------------
// --- [AGGREGATION] -----------------------------------------------------------
// --- [UNDO] ------------------------------------------------------------------
// --- [EXPORT] ----------------------------------------------------------------
```

**[TYPES] section:**
- `internal delegate void EventBatchFlushed(EventBatchSummary batch, Instant flushedAt)` — callback for when a batch is ready for WebSocket push. The caller (KargadanPlugin) provides this to bridge to EventPublisher.

**[CONSTANTS] section:**
- `private const int ChannelCapacity = 256`
- `private const int DebounceWindowMs = 200`
- `private static readonly BoundedChannelOptions ChannelOptions = new(ChannelCapacity) { FullMode = BoundedChannelFullMode.DropOldest, SingleWriter = false, SingleReader = true }`

**[LIFECYCLE] section:**
- Class: `internal sealed class ObservationPipeline : IDisposable`
- Constructor takes: `EventBatchFlushed onBatchFlushed, TimeProvider timeProvider`
- Fields: `Channel<RawDocEvent> _channel`, `System.Timers.Timer _debounceTimer`, `EventBatchFlushed _onBatchFlushed`, `TimeProvider _timeProvider`, `bool _disposed`
- `Start()` — subscribes to all 15 RhinoDoc events + Command.UndoRedo, starts the debounce timer
- `Stop()` — unsubscribes from all events, stops and disposes the timer, completes the channel writer
- `Dispose()` — calls Stop(), disposes channel reader/writer

**[SUBSCRIPTION] section — 15 RhinoDoc event handlers + subscribe/unsubscribe:**
Each handler must be FAST — write a RawDocEvent struct to `_channel.Writer.TryWrite(...)` and return immediately. No blocking, no JSON serialization, no WebSocket writes.

Subscribe to these 15 events (using the exact RhinoCommon event names):
1. `RhinoDoc.AddRhinoObject` — Type=ObjectsChanged, Subtype=Added, capture ObjectId + ObjectType + IsUndoRedo
2. `RhinoDoc.DeleteRhinoObject` — Type=ObjectsChanged, Subtype=Deleted, capture ObjectId
3. `RhinoDoc.UndeleteRhinoObject` — Type=ObjectsChanged, Subtype=Undeleted, capture ObjectId
4. `RhinoDoc.ReplaceRhinoObject` — Type=ObjectsChanged, Subtype=Replaced, capture ObjectId + OldObjectId
5. `RhinoDoc.ModifyObjectAttributes` — Type=ObjectsChanged, Subtype=Modified, capture ObjectId
6. `RhinoDoc.SelectObjects` — Type=SelectionChanged, Subtype=Selected
7. `RhinoDoc.DeselectObjects` — Type=SelectionChanged, Subtype=Deselected
8. `RhinoDoc.DeselectAllObjects` — Type=SelectionChanged, Subtype=DeselectAll
9. `RhinoDoc.LayerTableEvent` — Type=LayersChanged
10. `RhinoDoc.MaterialTableEvent` — Type=MaterialChanged
11. `RhinoDoc.DimensionStyleTableEvent` — Type=TablesChanged
12. `RhinoDoc.InstanceDefinitionTableEvent` — Type=TablesChanged
13. `RhinoDoc.LightTableEvent` — Type=TablesChanged
14. `RhinoDoc.GroupTableEvent` — Type=TablesChanged
15. `RhinoDoc.DocumentPropertiesChanged` — Type=PropertiesChanged

For handlers 1-5, detect IsUndoRedo via: `e.Document.UndoActive || e.Document.RedoActive` (confirmed boolean properties on RhinoDoc).

The Subscribe() and Unsubscribe() methods use `+=` and `-=` on each event. These are symmetric — every `+=` in Subscribe has a matching `-=` in Unsubscribe.

NOTE: `Command.EndCommand` is NOT subscribed here — ScriptRunner (in CommandExecutor) handles it per-call. `Command.UndoRedo` IS subscribed here — see [UNDO] section.

**[AGGREGATION] section:**
- `private void OnDebounceTimerElapsed(object? sender, System.Timers.ElapsedEventArgs e)` — drains the channel and builds EventBatchSummary
- `private Seq<RawDocEvent> DrainChannel()` — reads all available items from `_channel.Reader.TryRead()` into a Seq
- Build EventBatchSummary from drained events:
  - Group by `Type` (EventType) -> CategoryCount with Count
  - Within each CategoryCount, group by `Subtype` -> SubtypeCount
  - Set `ContainsUndoRedo = pending.Any(e => e.IsUndoRedo)`
  - Set `BatchWindowMs = DebounceWindowMs`
  - Set `TotalCount = pending.Count`
- Only invoke `_onBatchFlushed(batch, now)` if TotalCount > 0 (don't push empty batches)
- Use LanguageExt Seq operations: `.GroupBy()`, `.Map()`, `.ToSeq()` — NO LINQ `System.Linq` import
- `[BoundaryImperativeExemption(Reason = "ProtocolRequired", Ticket = "EXEC-03", Expiry = "2026-08-22")]` on the timer elapsed handler because System.Timers.Timer callback signature is `void (object?, ElapsedEventArgs)`.

**[UNDO] section:**
- Subscribe to `Command.UndoRedo` in Start(), unsubscribe in Stop()
- Handler: `private void OnUndoRedo(object? sender, UndoRedoEventArgs e)`
- Only emit on completion events (not begin events):
  - `e.IsEndUndo == true` — write RawDocEvent with Type=UndoRedo, Subtype=EventSubtype with a new `Undone` variant (or reuse existing if available)
  - `e.IsEndRedo == true` — write RawDocEvent with Type=UndoRedo, Subtype matching redo
  - All other cases (IsBeginUndo, IsBeginRedo, IsPurgeRecord, IsBeginRecording, IsEndRecording) — ignore
- Use switch expression with tuple pattern: `(e.IsEndUndo, e.IsEndRedo) switch { (true, _) => ..., (_, true) => ..., _ => ... }`
- `[BoundaryImperativeExemption(Reason = "ProtocolRequired", Ticket = "EXEC-03", Expiry = "2026-08-22")]` because Command.UndoRedo requires imperative event handler.

**C# Standards (INLINE):**
- `using static LanguageExt.Prelude;` at top
- File-scoped namespace: `namespace ParametricPortal.Kargadan.Plugin.src.observation;`
- Zero `var`, zero `if`/`else`, zero `for`/`foreach`
- Switch expressions for all branching
- Named parameters at all call sites
- Expression-bodied members for single-expression bodies
- All event handlers use `[BoundaryImperativeExemption]` with proper reason/ticket/expiry

**WebSocketHost.cs changes:**

1. Update `MessageDispatcher` delegate to support two-phase response:
   - Old: `Task<Fin<JsonElement>> (TransportMessageTag tag, JsonElement message, CancellationToken cancellationToken)`
   - New: `Task<Fin<JsonElement>> (TransportMessageTag tag, JsonElement message, Func<JsonElement, Task> sendAckAsync, CancellationToken cancellationToken)`
   - The `sendAckAsync` parameter allows the dispatcher to send an immediate acknowledgment before the final result

2. Update `DispatchByTagAsync` to pass a `sendAckAsync` delegate that writes to the active WebSocket:
   - Create a private method or inline lambda: `async Task SendAckAsync(WebSocket webSocket, JsonElement ackPayload)` that serializes and sends via `webSocket.SendAsync`
   - Pass this to `_dispatcher` in the DispatchByTagAsync call

3. Update `ProcessMessageAsync` to pass `sendAckAsync` through:
   - The sendAckAsync lambda captures the current `webSocket` reference from ProcessMessageAsync's scope
   - It serializes the ack JsonElement and sends it as a text WebSocket frame
   - The final result is still sent after BuildResponseAsync completes (existing behavior)

4. Ensure the `sendAckAsync` callback is safe: if the WebSocket is not open, the ack should be silently dropped (no exception propagation for ack failures)

**KargadanPlugin.cs changes:**

1. Add `ObservationPipeline` to BoundaryState:
   - `private readonly record struct BoundaryState(EventPublisher EventPublisher, SessionHost SessionHost, WebSocketHost WebSocketHost, ObservationPipeline ObservationPipeline)`

2. Update `DispatchMessageAsync` signature to match new MessageDispatcher:
   - Add `Func<JsonElement, Task> sendAckAsync` parameter
   - Pass it through to `DispatchCommandAsync`

3. Rewrite `DispatchCommandAsync` to route commands through CommandExecutor:
   - Accept the `sendAckAsync` parameter
   - Before marshaling to UI thread, send immediate ack: `await sendAckAsync(JsonSerializer.SerializeToElement(new { _tag = "command.ack", requestId = extractedRequestId }))`
   - Marshal to UI thread via ThreadMarshaler.RunOnUiThreadAsync
   - Inside the UI thread callback, determine execution mode based on CommandOperation:
     - If operation is `CommandOperation.ScriptRun` -> call `CommandExecutor.ExecuteScript(doc, scriptString, echo: true)` and wrap result into JsonElement
     - Otherwise -> call `CommandExecutor.ExecuteDirectApi(doc, envelope, handler, onUndoRedo)` where `handler` routes to the appropriate DocumentApi facade method based on the operation
   - Apply per-command timeout from `envelope.DeadlineMs` via `CancellationTokenSource.CreateLinkedTokenSource` + `CancelAfter`
   - The `onUndoRedo` callback should publish an event via EventPublisher to notify the harness of agent state changes

4. Create a private operation dispatch method that maps `CommandOperation` to the appropriate `CommandExecutor` facade call. Use a switch expression:
   - `CommandOperation.ObjectCreate` -> `CommandExecutor.AddObject(doc, geometry, attributes)` (deserialize from payload)
   - `CommandOperation.ObjectDelete` -> `CommandExecutor.DeleteObject(doc, objectId, quiet: true)`
   - `CommandOperation.ObjectUpdate` -> `CommandExecutor.ReplaceObject(doc, objectId, geometry)` or `CommandExecutor.ModifyAttributes`
   - `CommandOperation.ScriptRun` -> `CommandExecutor.ExecuteScript(doc, script, echo: true)`
   - Read operations -> execute without undo wrapping (just query and return)
   - For this phase, the actual payload deserialization and full operation dispatch can use a STUB approach: implement the dispatch structure with correct routing, but individual operation handlers can return `Fin.Fail("Operation not yet fully implemented")` for complex geometry operations. The critical path is: `ScriptRun` works end-to-end, `ObjectCreate`/`ObjectDelete`/`ObjectUpdate` work for basic cases, read operations work.

5. Wire ObservationPipeline lifecycle in OnLoad/OnShutdown:
   - In OnLoad (inside the `_state.Swap` block), create ObservationPipeline with a flush callback that wraps EventBatchSummary into EventEnvelope and publishes via EventPublisher
   - Call `observationPipeline.Start()` after creating it
   - In OnShutdown, call `observationPipeline.Stop()` and `observationPipeline.Dispose()` before disposing WebSocketHost
