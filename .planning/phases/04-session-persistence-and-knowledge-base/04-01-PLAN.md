    **Rewrite `checkpoint.ts` as PersistenceService:**

    Replace the existing `CheckpointService` with `PersistenceService` -- a single `Effect.Service` class at `kargadan/PersistenceService` using `effect: Effect.gen(...)`. Yield `SqlClient.SqlClient` and create a `Ref` for the write-through cache (in-memory store per user decision). The service exposes:

    1. `persist(input)` -- Atomic tool call + checkpoint write. Takes `{ toolCall: typeof KargadanToolCall.insert.Type, checkpoint: typeof KargadanCheckpoint.insert.Type, chatJson: string }`. Uses `sql.withTransaction(Effect.all([insertToolCall, upsertCheckpoint], { discard: true }))`. After transaction commits, update the in-memory Ref (write-through cache). Uses `SqlSchema.void` for both inserts. The checkpoint upsert uses `ON CONFLICT (session_id) DO UPDATE SET ...`.

    2. `hydrate(sessionId)` -- Silent resume with corruption fallback. Queries latest checkpoint via `SqlSchema.findOne`. If found, decode `loopState` via the LoopState schema. On decode failure: `Effect.logWarning('kargadan.checkpoint.corrupt', ...)`, return fresh state, preserve corrupted row (no delete). On success: return `{ fresh: false, state: loopState, chatJson: row.chatJson, sequence: row.sequence }`. On not found: return `{ fresh: true }`.

    3. `createSession(input)` -- Insert a new `kargadan_sessions` row with status `'running'`. Uses `SqlSchema.void` with `KargadanSession.insert`.

    4. `completeSession(input)` -- Update session status to `'completed'` or `'failed'`, set `endedAt`, update `toolCallCount`. Uses raw `sql` UPDATE.

    5. `listSessions(filter)` -- `SqlSchema.findAll` with optional status and date range filtering. Returns `ReadonlyArray<typeof KargadanSession.Type>`. Per user decision: filterable by status and date range, ordered by `started_at DESC`.

    6. `sessionTrace(sessionId)` -- `SqlSchema.findAll` returning tool calls for a session ordered by sequence. Per user decision: includes params, result, duration, status. Excludes raw LLM payloads.

    Keep `hashCanonicalState` and `verifySceneState` as module-level pure functions (exported). Delete `appendTransition`, `replay`, `save`, `restore`, `remove` -- they are replaced by the new methods.

    **Update `agent-loop.ts`:**

    Replace `CheckpointService` with `PersistenceService` throughout. The `persist` function inside the loop should now:

    - Build a `KargadanCheckpoint.insert.Type` record from the current loop state (serialized via JSON.stringify)

    The `appendTransition` calls throughout the loop (plan, execute, persist stages) must each be converted to `persistence.persist` calls with appropriate tool call records. Each call is an atomic checkpoint save per the user's locked decision ("every tool call triggers a checkpoint save").
