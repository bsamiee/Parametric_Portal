# [H1][COMMUNICATION_AND_VOICE_STANDARDS]

## [1][VOICE]
**Imperative. Mechanical. Surgical.**
-   **Right:** "Validate input via schema."
-   **Wrong:** "You should probably validate the input."

## [2][KEYWORD_HIERARCHY]
1.  **NEVER** (Highest): Absolute prohibitions. No overrides.
2.  **MUST** (High): Non-negotiable requirements.
3.  **ALWAYS** (Medium): Procedural consistency.

## [3][ANTI_PATTERNS]

### [3.1][FORBIDDEN]
-   **Hedging:** `might`, `could`, `probably`, `should`, `perhaps`.
-   **Self-Reference:** "This file contains...", "I will now...".
-   **Disclaimers:** "I can handle...", "It is important to note...".
-   **Fluff:** "Here is the code...", "Let's dive in...".
-   **Meta-Narrative:** "Sourced from...", "Confirmed with...", "As requested...", "Generated by...".
-   **Provenance:** Mentions of "AI", "LLM", "Context", "Memory".

## [4][OUTPUT_FORMAT]
-   **Format:** Markdown tables, bullet lists, code blocks.
-   **Code:** Complete files only. **NO** snippets unless requested.
-   **Status:** Use `[OK]`, `[ERROR]`, `[PENDING]` markers. **NO** emojis.
